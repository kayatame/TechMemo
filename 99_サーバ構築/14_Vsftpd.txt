○ パッケージインストール
# dnf install vsftpd
# rpm -q vsftpd


○ 設定ファイル調整
# cp -ip /etc/vsftpd/vsftpd.conf{,.org}
# vi /etc/vsftpd/vsftpd.conf
-----------------------------------------------------------------------------
local_umask=002
xferlog_file=/var/log/xferlog
dual_log_enable=YES
ascii_upload_enable=YES
ascii_download_enable=YES
chroot_local_user=NO
chroot_list_enable=YES
chroot_list_file=/etc/vsftpd/chroot_list
allow_writeable_chroot=YES
listen=YES
listen_ipv6=NO
userlist_deny=NO
use_localtime=YES
force_dot_files=YES
-----------------------------------------------------------------------------
# diff /etc/vsftpd/vsftpd.conf{,.org}

RHEL8までであれば、vsftpd に関しては xinetdでも起動も可能  
下記を設定することでvsftpdをListenして起動する
-----------
listen=YES
-----------

下記設定にてchrootに関して設定するので注意
---------------------------------------------------------------------------------------------------------------------------
|                       |                  chroot_list_enable=YES                   |          chroot_list_enable=NO      |
| --------------------- | --------------------------------------------------------- | ----------------------------------- |
| chroot_local_user=YES | chroot_listで指定したファイルにリストされたユーザ以外はchroot | すべてのローカルユーザはchrootされる   |
| chroot_local_user=NO  | chroot_listで指定したファイルにリストされたユーザはchroot　   | すべてのローカルユーザはchrootされない |
---------------------------------------------------------------------------------------------------------------------------


○ chroot_list / user_list調整
# vi /etc/vsftpd/chroot_list
-----------------------------------------------------------------------------
chrootを設定するユーザを記載
-----------------------------------------------------------------------------

# cp -ip /etc/vsftpd/user_list{,.org}
# vi /etc/vsftpd/user_list
-----------------------------------------------------------------------------
接続を許可するユーザを記載
-----------------------------------------------------------------------------
# diff /etc/vsftpd/user_list{,.org}

「userlist_deny=NO」で設定しているので、user_listはホワイトリスト扱いになる


○ サービス起動
# systemctl --now enable vsftpd
# systemctl status vsftpd


○ ファイルアップロードテスト
# tail /var/log/xferlog
